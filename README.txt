ЦЕ НОВА ВЕРСІЯ (пункт 4):
- У закритому стані всі квадрати разом виглядають як одне цілісне полотно (фон).
- Квадрати "вриваються" переворотом і показують слова.
- Потім повертаються назад у полотно.
- Між сценами є красивий наплив (кросфейд) з коротким зниканням плиток.

ЩО РЕДАГУВАТИ
- Тільки config.js

НАЛАШТУВАННЯ FRONT DARK (контур/затемнення на закритих квадратах)
- За замовчуванням вимкнено: frontDarkDefault: 0
- Для окремого квадрата в сцені додайте: frontDark: 0.20

БЛЮР ПІД ЧАС ПЕРЕХОДУ
- timing.sceneBlurPx керує блюром під час напливу (0 = без блюру)


МʼЯКА ОБВОДКА / GLOW (глобально)
- У config.js: frontGlowDefault: 0 (вимкнено)
- Наприклад увімкнути: frontGlowDefault: 0.18
- Інтенсивність/мʼякість: frontGlowBlurPx, frontGlowSpreadPx, frontGlowInsetPx


РАМКА/КОНТУР НАВКОЛО ПЛИТОК
- У config.js: tileFrameAlpha: 0 і tileFrameSizePx: 0 (за замовчуванням ВИМКНЕНО)
- Якщо потрібен контур: tileFrameAlpha: 0.22, tileFrameSizePx: 12


ЩО ТАКЕ "СЛІДИ" КВАДРАТІВ
- Це тінь (drop-shadow) від плиток навіть у закритому стані.
- У config.js вимкнено за замовчуванням: tileShadowAlpha: 0
- Якщо потрібна глибина: tileShadowAlpha: 0.25..0.45

ПАУЗА ПЕРЕД ПЕРШИМ ПЕРЕВОРОТОМ
- timing.initialDelayMs: 3000 означає, що після старту сцени/переходу чекаємо 3 секунди і тільки тоді починаємо перевороти.


АБСОЛЮТНО НЕВИДИМІ ПЛИТКИ У ЗАКРИТОМУ СТАНІ
- У config.js: closedTilesInvisible: true
- Тоді в стані front плитки прозорі (видно тільки фон), а перед переворотами вони автоматично плавно зʼявляються.
- Швидкість появи: closedTilesFadeMs

ПАУЗА ПЕРЕД ПЕРЕХОДОМ
- timing.holdFrontBetweenScenesMs: 3000 — тримаємо закрите полотно 3 секунди перед напливом на наступну сцену.

============================================================
НАЛАШТУВАННЯ У config.js (коротко і зрозуміло)
============================================================

1) timing (таймінги)
- initialDelayMs
  Пауза після старту сцени / після переходу на нову сцену перед першим переворотом.

- holdTileMs
  Для кожного квадрата: EN тримаємо X мс → UA тримаємо X мс → наступний квадрат.

- holdFinalMs
  Після останнього квадрата: ВСІ квадрати EN X мс → ВСІ квадрати UA X мс.

- holdFrontBetweenScenesMs
  Після фінального UA повертаємо всі квадрати у «закритий» стан (полотно) і тримаємо його X мс перед переходом.

- sceneFadeMs
  Тривалість напливу (кросфейд) між сценами. Рекомендація: 1200–1600.

- sceneBlurPx
  Блюр під час напливу (0 = без блюру). Рекомендація: 8–14.

2) closedTilesInvisible (глобально)
- closedTilesInvisible: true
  У закритому стані плитки повністю прозорі (видно лише фон).
  Перед переворотами плитки автоматично плавно зʼявляються.

- closedTilesFadeMs
  Швидкість появи/зникання плиток (мс), коли closedTilesInvisible=true.

3) Рамка / тінь / glow (що за що відповідає)
- tileFrameAlpha + tileFrameSizePx
  Тонкий контур/рамка навколо плиток (не glow). За замовчуванням вимкнено.

- tileShadowAlpha (+ tileShadowYpx, tileShadowBlurPx)
  Тінь від плиток (це і є «сліди»). За замовчуванням вимкнено.

- frontGlowDefault (+ frontGlowBlurPx, frontGlowSpreadPx, frontGlowInsetPx)
  Мʼякий glow-обвід (за замовчуванням вимкнено).


ПЛАВНЕ ПРОЯВЛЕННЯ/РОЗЧИНЕННЯ ПЛИТОК (щоб уникнути "мигання")
- tilesIntroFadeMs: скільки мс плитки плавно проявляються на початку сцени (поки йде initialDelayMs)
- tilesOutroFadeMs: скільки мс плитки плавно розчиняються наприкінці сцени перед переходом
- tilesMinOpacity: мінімальна непрозорість (0 = повністю зникли, 0.12 = легкий "слід")
- tilesMaxOpacity: максимальна непрозорість (зазвичай 1)


ПРОБЛЕМА "ЗАЛИШКУ" ПОПЕРЕДНЬОЇ СЦЕНИ У ПЛИТКАХ
- Якщо tilesMinOpacity > 0, під час переходу між сценами плитки можуть ледь показувати вирізку старого фону.
- Рішення: transitionTilesMinOpacity: 0.0 (плитки повністю розчиняються саме під час переходу).
