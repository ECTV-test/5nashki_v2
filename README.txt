Flip Tiles — Scenes (Canvas Reveal v4)

РЕДАГУЙТЕ ТІЛЬКИ config.js.

============================================================
РЕКОМЕНДОВАНІ РОЗМІРИ ЗОБРАЖЕНЬ
============================================================
- Фон (bg): 1920×1080 (найкраще) / 2560×1440 (дуже чітко) / 3840×2160 (4K, важче для завантаження)
- Картинки для квадратів (front): 800×800 або 1024×1024 (найкраще). Мінімум 600×600

Примітка:
- У режимі “полотно” (коли front не задано) плитки беруть “вирізку” з bg — тому якість bg критична.

============================================================
ОРГАНІЗАЦІЯ ФАЙЛІВ (щоб не плутатися)
============================================================
- Кожну сцену зберігайте в окремій папці:
  assets/scene01/bg.jpg
  assets/scene01/t1.jpg, t2.jpg, t3.jpg, b1.jpg, b2.jpg   (опційно)
  assets/scene02/bg.jpg
  assets/scene02/t1.jpg, t2.jpg, t3.jpg, b1.jpg, b2.jpg   (опційно)

- Те, що повторюється між сценами, кладіть у:
  assets/shared/
  і підставляйте шлях у сцені.

============================================================
ПОЛОТНО vs ОКРЕМІ КАРТИНКИ НА ПЛИТКИ (front)
============================================================
- За замовчуванням (front НЕ задано):
  Плитка у закритому стані показує “вирізку” з bg → виглядає як одне цілісне полотно.

- Якщо задати front для плитки:
  Плитка у закритому стані показує вашу картинку (cover + center), а не вирізку з bg.
  Приклад:
    t1: { front:"./assets/scene01/t1.jpg", text:{ en:"...", ua:"..." } }

============================================================
ТАЙМІНГИ (timing)
============================================================
- timing.initialDelayMs:
  Пауза після старту сцени / після переходу на нову сцену перед першим переворотом.

- timing.holdTileMs:
  Для КОЖНОГО квадрата: EN X мс → UA X мс → наступний квадрат.

- timing.holdFinalMs:
  Після останнього квадрата: ВСІ квадрати EN X мс → ВСІ квадрати UA X мс.

- timing.holdFrontBetweenScenesMs:
  Після фінального UA повертаємо квадрати у “закритий” стан (front/полотно)
  і тримаємо закрите полотно X мс перед переходом на наступну сцену.

- timing.sceneFadeMs:
  Тривалість “напливу” (кросфейд) між сценами. Рекомендація: 1200–1600.

- timing.sceneBlurPx:
  Блюр під час напливу (0 = без блюру). Рекомендація: 8–14.

============================================================
ШВИДКОСТІ (для кожного квадрата окремо)
============================================================
- flipToBackMs:
  Швидкість першого перевороту (front → EN/back).

- flipTextMs:
  Швидкість перевороту тексту на звороті (EN ↔ UA) БЕЗ повернення до front.

============================================================
ПЛАВНЕ ПРОЯВЛЕННЯ/РОЗЧИНЕННЯ ПЛИТОК (щоб уникнути “мигання”)
============================================================
- tilesIntroFadeMs:
  Скільки мс плитки плавно проявляються на початку сцени (поки йде initialDelayMs).

- tilesOutroFadeMs:
  Скільки мс плитки плавно розчиняються в кінці сцени перед переходом.

- tilesMinOpacity:
  Мінімальна непрозорість під час розчинення (0 = повністю зникли, 0.12 = легкий “слід”).

- tilesMaxOpacity:
  Максимальна непрозорість (зазвичай 1).

- transitionTilesMinOpacity:
  Мінімальна непрозорість САМЕ під час переходу між сценами.
  Рекомендація: 0, щоб не було “залишків” попереднього фону у плитках.

============================================================
РАМКА / ТІНЬ / GLOW (що за що відповідає)
============================================================
- tileFrameAlpha + tileFrameSizePx:
  Тонкий контур/рамка навколо плиток (не glow). За замовчуванням вимкнено.

- tileShadowAlpha (+ tileShadowYpx, tileShadowBlurPx):
  Тінь від плиток (це і є “сліди”). Можна ввімкнути легенько, щоб не було відчуття “мигання”.

- frontGlowDefault (+ frontGlowBlurPx, frontGlowSpreadPx, frontGlowInsetPx):
  Мʼякий glow-обвід (за замовчуванням вимкнено).

============================================================
КЕРУВАННЯ / HUD
============================================================
- Автозапуск при відкритті сторінки
- Тап/клік по екрану перезапускає анімацію (коли HUD прихований)
- HUD прихований за замовчуванням:
  - подвійний тап/подвійний клік у ЛІВОМУ ВЕРХНЬОМУ куті показує/ховає HUD
